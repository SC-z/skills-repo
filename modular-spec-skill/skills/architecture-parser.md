# 子 Skill：架构语义解析

## 输入
- 整体架构描述
- 目标模块名称 / 路径
- （可选）模块与其他组件的已知关系

## 分析目标
1. 确认模块所属子系统
2. 明确模块在整体架构中的角色
3. 识别上下游模块与依赖方向
4. 提取与模块相关的架构约束

## 输出
- 模块架构定位说明
- 上游 / 下游模块列表
- 关键依赖与依赖原因
- 架构级约束摘要

## 输出要求
- 禁止基于源码猜测架构意图
- 若架构信息不足，需显式标注不确定性
- 输出用于后续子 Skill 复用，不直接生成 Spec
