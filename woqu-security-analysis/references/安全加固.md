pam_pwquality.so

### 2. 访问控制
#### 2.1 hosts.allow和hosts.deny(`可以修改`)
> /etc/hosts.allow和hosts.deny文件如果openssh版本比较高，7.x或者8.x的话，是不生效的。

也是linux的黑白名单，hosts.allow中主机允许通过，hosts.deny中主机则被禁止登录

> 修改/etc/hosts.allow文件，允许210.13.218.*所有的机器都可以通过ssh登录主机

```
#
# hosts.allow This file describes the names of the hosts which are
# allowed to use the local INET services, as decided
# by the ‘/usr/sbin/tcpd’ server.
#
sshd:210.13.218.*:allow
sshd:222.77.15.*:allow
```

禁止登录的黑名单

> /etc/hosts.deny文件，将所有的主机都禁止sshd登录

```
#
# hosts.deny This file describes the names of the hosts which are
# *not* allowed to use the local INET services, as decided
# by the ‘/usr/sbin/tcpd’ server.
#
# The portmap line is redundant, but it is left to remind you that
# the new secure portmap uses hosts.deny and hosts.allow. In particular
# you should know that NFS uses portmap!
sshd:all:deny
```

### 7. 网络通信
#### 7.1 禁止ICMP重定向（`可以修改`)
> /etc/sysctl.conf
```
net.ipv4.conf.all.accept_redirects=0
```
防止icmp重定向欺骗攻击：icmp重定向是为了优化路由

#### 7.2 禁止IP源路由（`可以修改`）
> /proc/sys/net/ipv4/conf/*/accept_source_route

允许 IP 源路径路由(IP source routing)会使得黑客能够欺骗你的计算机，截取信息包

#### 7.3 更改主机解析地址的顺序（`可以修改`）
> /etc/host.conf
```
order hosts，bind
```
首先通过DNS解析IP地址，然后通过hosts文件解析
#### 7.4 修改iptables（以下`可以修改`）
> iptables

ICMP timestamp请求相应漏洞和允许Traceroute探测请求
```
iptables -A INPUT -p ICMP --icmp-type timestamp-request -j DROP
iptables -A INPUT -p ICMP --icmp-type timestamp-reply -j DROP
iptables -A INPUT -p ICMP --icmp-type time-exceeded -j DROP
iptables -A OUTPUT -p ICMP --icmp-type time-exceeded -j DROP
```

检测到远端RPCBIND/PORTMAP正在运行中（CVE-1999-0632）这个问题的
```
iptables -I INPUT -p tcp -m tcp --dport 111 -j DROP
iptables -I INPUT -s 10.102.0.41 -p tcp --dport 111 -j ACCEPT
```
#### 7.5  禁止ssh agent转发（`可以修改`）
开启ssh agent转发后可以通过一系列操作，直接免密登录内网系统
> /etc/ssh/sshd_config
```
AllowAgentForwarding no
```
### 8.内核参数
#### 8.1 sysctl.conf
>/etc/sysctl.conf
```
vm.swappiness =1
kernel.numa_balancing = 0
```
vm.swappiness是使用swap空间的积极程度，oracle官方是这么推荐的
> To achieve the same behavior of swappiness as previous versions of Red Hat Enterprise Linux 6.4 in which the recommendation was to set swappiness to 0, set swappiness to the value of 1. The recommendation of swappiness for Red Hat Enterprise Linux 6.4 or higher running Oracle databases is now the value of 1.

kernel.numa_balancing是关闭numa，在产品中也会在bios中关闭numa的
对于产品中sysctl.conf中配置的参数，都`不建议修改`，因为这些参数都是最佳的

### 11.其他
* 文件权限：一般会修改的日志文件的权限，权限一般会变大一些，但是other的权限还是0，这个`可以修改`。
* 关闭core dump操作：方便出问题后排查问题，`不建议修改`
* chattr：只允许追加，不允许修改和移动等操作。`不建议修改`，因为message切片时会使用mv，这样会导致并不能切片，message日志越来越大
* 主机名：由于产品在安装时将主机名保存至数据库和hosts中，修改后使用可能会出现问题。故`不建议修改`
* 安全补丁（安装最新的OS补丁）：`禁止`，影响一体机稳定
* 开启主机的snmp服务：`可以修改`，不要开启防火墙
