# 子 Skill：核心流程与实现映射

## 输入
- 模块源码
- 架构语义解析结果
- 模块职责定义

## 分析目标
1. 提炼模块的核心执行流程
2. 识别关键抽象对象与数据结构
3. 映射实现细节与架构设计意图
4. 标注实现中与设计不一致的部分

## 输出
- 核心流程高层描述
- 关键接口 / 数据结构说明
- 实现与设计意图的对应关系
- 潜在风险点提示

## 输出要求
- 避免逐行代码解释
- 重点描述“为什么这样实现”
- 若发现实现偏离设计，需明确指出
