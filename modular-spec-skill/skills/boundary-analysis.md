# 子 Skill：边界与职责分析

## 输入
- 模块接口定义
- 模块源码
- 架构语义解析结果

## 分析目标
1. 明确模块对外暴露的能力
2. 识别模块内部与外部的职责分界
3. 发现可能的职责侵蚀或耦合风险
4. 提供边界收敛建议

## 输出
- 模块职责边界说明
- 对外依赖与被依赖关系
- 边界模糊或异常点
- 改进建议（如适用）

## 输出要求
- 边界判断以架构定义优先
- 不对业务正确性做判断
- 重点服务于重构与接手场景
