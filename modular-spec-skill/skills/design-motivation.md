# 子 Skill：设计动机生成

## 输入
- 模块职责描述
- 架构语义解析结果
- （可选）历史设计背景

## 分析目标
1. 解释模块产生的背景问题
2. 明确该模块解决了什么、避免了什么
3. 描述核心设计取舍及原因
4. 标注被放弃的替代方案（若可推断）

## 输出
- 设计背景与问题域说明
- 设计目标与非目标
- 关键设计动机
- 设计取舍总结

## 输出要求
- 以架构目标为第一依据
- 禁止事后合理化代码实现
- 若设计动机无法从输入推导，需明确说明
